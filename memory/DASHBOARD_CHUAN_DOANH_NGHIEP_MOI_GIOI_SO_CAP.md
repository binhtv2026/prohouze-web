# DASHBOARD CHUẨN CHO DOANH NGHIỆP MÔI GIỚI BẤT ĐỘNG SẢN SƠ CẤP

## Mục tiêu

Đây là bảng chốt chuẩn để thiết kế lại dashboard theo đúng nghiệp vụ của doanh nghiệp môi giới bất động sản sơ cấp.

Nguyên tắc dùng bảng này:

1. Mỗi dòng là một cụm công việc thật.
2. Mỗi tab cháu phải dẫn tới nơi xử lý việc, không phải chỉ xem.
3. Không trộn vai trò kinh doanh hiện trường với back office.
4. Dashboard chỉ giữ thứ giúp người dùng làm việc nhanh hơn, chốt khách tốt hơn, kiểm soát vận hành tốt hơn.
5. Nếu một mục không trả lời được câu hỏi “người dùng vào đây để làm gì”, mục đó phải bỏ.

## Kết luận chốt

### 1. Dashboard nào là bắt buộc

- Ứng dụng kinh doanh
- Ứng dụng cộng tác viên / đại lý
- Giao diện web quản lý kinh doanh
- Giao diện web lãnh đạo
- Giao diện web marketing
- Giao diện web tài chính
- Giao diện web nhân sự
- Giao diện web pháp lý
- Giao diện web quản trị website / nội dung
- Giao diện web quản trị hệ thống

### 2. Những gì phải bỏ khỏi dashboard hiện tại

- Ghi chú dev note
- Màn chỉ để xem không có hành động
- Tab trùng ý
- Tab không gắn công việc thật
- Trộn marketing với quản trị website
- Trộn nhân sự tổng quát vào màn sale
- Trộn back office sâu vào ứng dụng kinh doanh

### 3. Quy tắc làm lại UI

- Mỗi dashboard tổng chỉ trả lời tối đa 5 câu hỏi quan trọng nhất của user
- Tab cha = nhóm công việc
- Tab con = nhóm chức năng
- Tab cháu = trạng thái / phân loại / hành động
- Click từ tab cháu phải ra nơi xử lý công việc
- Ứng dụng ưu tiên tốc độ và hành động nhanh
- Giao diện web ưu tiên phân tích sâu, bảng nhiều cột, duyệt và kiểm soát
